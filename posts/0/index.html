OpenGL 核心对象：VAO、VBO 与 EBO
💡 核心类比：

VBO = 装满原材料（顶点数据）的箱子

EBO = 按数字连线的图纸（索引）

VAO = 画家的记忆档案（记住用哪个箱子、看哪张图纸、怎么解析数据）

  1. VBO (Vertex Buffer Object) | 顶点缓冲对象
    本质：在 GPU 显存中开辟的内存区域。

  • 存储内容：真金白银的顶点数据（坐标 x,y,z，颜色，纹理坐标等）。

  • 核心作用：将大量数据一次性从 CPU 传到 GPU。避免早期 OpenGL 中 CPU 每画一个点就发送一次数据的极低效率（减少 CPU/GPU 通信开销）。

  1. EBO (Element Buffer Object) | 索引缓冲对象
    本质：在 GPU 显存中开辟的内存区域。

  • 存储内容：顶点数据的索引（纯数字），指向 VBO 中的顶点。

  • 核心作用：指示画图的连接顺序。通过复用共用的顶点（如两个三角形共用一条边），==极大地节省显存空间==和处理时间。

  1. VAO (Vertex Array Object) | 顶点数组对象
    本质：一个状态追踪器（配置档案）。==VAO 本身不存储任何顶点数据！==

  • 存储内容：记录了以下状态：

    1. VBO 解析规则（通过 glVertexAttribPointer 设置的步长、偏移量等属性）。
    2. 当前绑定的 EBO。

  • 核心作用：OpenGL 是状态机，VAO 把繁琐的“数据解析配置”打包记下来。渲染时只需绑定对应的 VAO，GPU 就知道该怎么画了。
